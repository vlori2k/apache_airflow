1. installer airflow-pakken:
- pip install 'apache-airflow==3.2.0' --constraint "https://raw.githubusercontent.com/apache/airflow/constraints-3.2.0/constraints-3.10.txt"

vanlig Airflow fungerer ikke direkte på Windows, men det går fint siden vi kjører med Docker lokalt.

2. hent ferdig docker-compose.yaml fra Apache Airflow:
- windows: Invoke-WebRequest -Uri "https://airflow.apache.org/docs/apache-airflow/3.2.0/docker-compose.yaml" -OutFile "docker-compose.yaml"
- linux: curl -L "https://airflow.apache.org/docs/apache-airflow/3.2.0/docker-compose.yaml" -o docker-compose.yaml

3. lag .env-fila automatisk:
- Set-Content -Path ".env" -Value "AIRFLOW_UID=50000`

4. initialiser Airflow-oppsettet:
- docker compose up airflow-init

5. start Airflow:
- docker compose up

6. åpne Airflow i nettleseren:
- http://localhost:8080

login:
- brukernavn: airflow
- passord: airflow

7. for å slå av:
- docker compose down -v



ekstra kommandoer:

- docker compose up -d --no-deps --build airflow-apiserver airflow-scheduler'

hva gjør dette?:
bygg opp igjen og start bare airflow-apiserver og airflow-scheduler, i bakgrunnen, uten å starte avhengige tjenester automatisk.

Del for del:
docker compose up
starter tjenester fra docker-compose.yaml

-d
kjør dem i bakgrunnen, detached mode

--no-deps
ikke start andre services som disse vanligvis er avhengige av, som for eksempel Postgres eller Redis

--build
bygg imaget på nytt før oppstart, for eksempel hvis du har endret Dockerfile
airflow-apiserver airflow-scheduler
bare disse to tjenestene skal startes

Så i praksis:
“Bygg på nytt og start bare Airflow API-server og scheduler i bakgrunnen.”
